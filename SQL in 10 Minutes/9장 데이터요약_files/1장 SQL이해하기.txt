1) 데이터베이스 
- 정리된 데이터를 저장하기 위한 그릇
- 데이터베이스 소프트웨어는 데이터베이스 관리 시스템(DBMS : DataBase Management System)이고, 데이터베이스는 이 DBMS를 생성하고 조작하는 물리적인 저장공간을 뜻함.
- SQL : DBMS와 소통하기 위해 필요함
- 대표적인 DBMS : Oracle, Db2, SQL Server, Postgre SQL, MySQL, SQLite 등


2) 테이블(Table)
- 특정한 형태의 데이터로 이루어진 구조화된 목록
- 테이블에 저장하는 데디터는 한 가지 종류거나 목록이어야 함


3) 스키마(Schema)
- 데이터베이스, 테이블의 구조와 속성에 대한 정보의 집합


4) 열(Column, 필드) ↔ 행(Row, 레코드)
- 테이블에 있는 하나의 필드. 모든 테이블은 한 개 이상의 열로 구성되어 있음


5) 데이터나누기
- 데이터를 나누는 것은 데이터베이스를 설계하는 시점에서 서비스의 요구사항에 따라 달라질 수 있다.


6) 데이터형(Datatype)
- 열에 저장할 수 있응 데이터의 유형을 정의한 것
- 모든 테이블 열에는 관련된 데이터형이 있는데 특정 데이터를 제한하거나 허용함 ex) 수치형 열에는 문자가 들어갈 수 없음
- 데이터형과 데이터형의 이름은 서로 다른 SQL 소프트웨어가 호환되지 않게 하는 주요 원인 중 하나이기 때문에 테이블 스키마를 생성할 때 항상 데이터형을 고려해야 함


7) 행(Row, 레코드)
- 테이블에 있는 레코드


8) 기본 키(Primary Key)
- 테이블에 있는 각 행을 구별 짓는 열(또는 열 집합)
- 두 행이 같은 기본 키 값을 가질 수 없음
- 모든 행은 기본 키 값을 가져야 함(즉, 기본 키 열은 NULL 값을 허용하지 않음)
- 기본 키 열에 있는 값은 변경하거나 업데이트할 수 없음
- 기본 키 값은 재사용할 수 없음(테이블에서 어떤 행을 삭제할 경우 그 기본 키도 삭제되지만, 다른 행에 다시 할당할 수 없음)


9) SQL (구조화된 질의 언어, Structured Query Language)
- 데이터베이스와 소통하기 위해 고안된 언어
- 특정 데이터베이스 회사가 독점하는 언어가 아니기 때문에 이 언어를 익히면 거의 모든 데이터베이스를 사용할 수 있음
- 배우기 쉬움
- 명령어를 적절하게 조합하면 정교한 데이터베이스 작업을 수행할 수 있음


10) ANSI (American National Standards Institute)
- 표준 SQL
